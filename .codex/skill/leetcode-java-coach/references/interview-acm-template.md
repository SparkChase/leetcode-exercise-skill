# Java 大厂笔试 ACM 出题模板

当用户明确要求“大厂笔试风格”或“只给题目, 我自己写”时, 优先使用下面的输出结构。

## 目标

把题目组织成常见在线笔试风格, 让用户直接按 Java ACM/OJ 方式实现。

默认要求:

- 语言: Java
- 入口类: `Main`
- 不写 `package`
- 从标准输入读取
- 向标准输出打印
- 默认不给完整题解和代码

## 标准输出结构

1. 题目名称
2. 题目描述
3. 输入描述
4. 输出描述
5. 数据范围
6. 示例 1
7. 示例 2
8. 提示
9. 开始写吧

## 推荐写法

### 题目名称

用简洁中文标题, 必要时补一行英文别名。

### 题目描述

- 尽量像真实笔试题, 不是只给一个裸算法标签
- 可以有轻量业务包装, 但不要喧宾夺主
- 必须清楚说明需要计算什么

### 输入描述

- 明确第一行、第二行分别表示什么
- 如果有多组数据, 必须写清楚
- 如果读到 EOF 结束, 也要明确写出

### 输出描述

- 明确每组数据输出什么
- 多组数据时说明是否每行一个结果

### 数据范围

- 数组长度范围
- 数值范围
- 时间复杂度期望
- 空间复杂度期望

### 示例

每个示例都包含:

- 输入
- 输出
- 说明

### 提示

只给少量提醒, 不要直接泄露完整解法:

- 注意边界条件
- 注意是否有重复元素
- 注意是否需要 `long`
- 注意时间复杂度是否会超时

## 输出禁忌

默认不要直接输出:

- 完整 Java 代码
- 完整伪代码
- 逐步题解
- 关键不变量的直接答案

除非用户明确要求:

- 给提示
- 帮我检查思路
- 帮我 debug
- 直接给题解

## 示例骨架

```text
题目名称: 两数之和

题目描述:
给定一个长度为 n 的整数数组 nums 和一个目标值 target, 请你在数组中找到两个数, 使得它们的和等于 target。
如果存在多组答案, 输出任意一组下标即可。下标从 0 开始。

输入描述:
第一行输入两个整数 n 和 target。
第二行输入 n 个整数, 表示数组 nums。

输出描述:
输出两个整数 i 和 j, 表示满足 nums[i] + nums[j] = target 的下标。

数据范围:
1 <= n <= 2 * 10^5
-10^9 <= nums[i], target <= 10^9

示例 1:
输入:
4 9
2 7 11 15

输出:
0 1

说明:
因为 nums[0] + nums[1] = 9。

提示:
请先自己完成 Java ACM 写法, 默认使用 Main 类和标准输入输出。
```
